# 部署配置指南

## 腾讯云自动部署配置

本指南介绍如何使用 GitHub Actions 配置自动部署到腾讯云服务器的完整流程。

### 前提条件

1. **腾讯云服务器**需要具备：
   - 已启用 SSH 访问
   - 已安装 Python 3.11+
   - 已安装 PM2（用于进程管理）或其他进程管理器
   - 已安装 Git
   - 必要的系统依赖

2. **GitHub 仓库 Secrets** - 需要在 GitHub 仓库设置中配置：

### 必需的 GitHub Secrets

| 密钥名称 | 描述 | 示例 |
|---------|------|------|
| `TENCENT_SERVER_HOST` | 腾讯云服务器的 IP 地址或域名 | `123.456.789.012` |
| `TENCENT_SERVER_USER` | 服务器的 SSH 用户名 | `ubuntu` 或 `root` |
| `TENCENT_SERVER_SSH_KEY` | 服务器访问的 SSH 私钥 | `-----BEGIN OPENSSH PRIVATE KEY-----...` |
| `TENCENT_PROJECT_PATH` | （可选）服务器上项目部署路径 | `/app/ai-paper-writer` |
| `TENCENT_SERVER_PORT` | （可选）SSH 端口，默认为 22 | `22` |
| `APP_PORT` | （可选）应用端口，用于健康检查 | `8000` |

### API 密钥和环境变量

| 密钥名称 | 描述 |
|---------|------|
| `DASHSCOPE_API_KEY` | DashScope LLM 服务 API 密钥 |
| `TAVILY_API_KEY` | Tavily 搜索服务 API 密钥 |
| `DATABASE_URL` | 生产环境 PostgreSQL 连接 URL |

### 服务器设置

#### 1. 初始服务器配置

```bash
# 连接到腾讯云服务器
ssh username@your-server-ip

# 更新系统包
sudo apt update && sudo apt upgrade -y

# 安装必要的依赖
sudo apt install -y python3 python3-pip python3-venv git postgresql postgresql-contrib

# 安装 PM2 进程管理器（需要 Node.js）
curl -fsSL https://deb.nodesource.com/setup_16.x | sudo -E bash -
sudo apt install -y nodejs
sudo npm install -g pm2
```

#### 2. 项目目录设置

```bash
# 创建项目目录
sudo mkdir -p /app/ai-paper-writer
sudo chown $USER:$USER /app/ai-paper-writer

# 克隆仓库
cd /app/ai-paper-writer
git clone your-repo-url .
```

#### 3. 数据库设置（PostgreSQL）

```bash
# 切换到 postgres 用户
sudo -u postgres psql

# 创建数据库和用户
CREATE DATABASE ai_paper_writer;
CREATE USER app WITH PASSWORD 'your-secure-password';
GRANT ALL PRIVILEGES ON DATABASE ai_paper_writer TO app;

# 退出 psql
\q
```

#### 4. Python 环境设置

```bash
# 创建虚拟环境（推荐）
python3 -m venv venv
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt
```

### GitHub Actions 配置

工作流程将自动执行：

1. **构建和测试** 应用程序
2. **部署到腾讯云服务器** 当推送版本标签时通过 SSH
3. **执行健康检查** 验证部署成功

### 部署流程

#### 手动触发部署

创建版本标签来触发部署：

```bash
# 创建并推送版本标签
git tag v1.0.0
git push origin v1.0.0
```

#### 部署过程中发生的事情

1. **代码检出**：检出最新代码
2. **版本检测**：从标签中提取版本号
3. **Docker 镜像构建**：构建生产环境 Docker 镜像并推送到 GitHub 容器注册表
4. **SSH 部署**：通过 SSH 连接到腾讯云服务器
5. **代码更新**：从仓库拉取最新代码
6. **环境设置**：更新环境变量
7. **服务管理**：停止现有服务并启动新版本
8. **健康检查**：验证服务正常运行

### 监控和故障排除

#### 检查部署状态

```bash
# 检查 PM2 状态
pm2 status
pm2 logs ai-paper-writer

# 检查应用日志
tail -f app.log

# 检查系统日志
journalctl -u ssh
```

#### 常见问题及解决方案

1. **SSH 连接失败**
   - 验证 GitHub Secrets 中的 SSH 密钥配置正确
   - 检查服务器防火墙设置
   - 确保服务器上 SSH 服务正在运行

2. **依赖安装失败**
   - 检查 Python 版本兼容性
   - 验证服务器网络连接
   - 检查可用磁盘空间

3. **应用无法启动**
   - 检查应用日志：`pm2 logs ai-paper-writer`
   - 验证环境变量设置正确
   - 检查所需端口是否可用

4. **数据库连接问题**
   - 验证 DATABASE_URL 格式
   - 检查 PostgreSQL 服务状态
   - 验证数据库用户权限

### 安全最佳实践

1. **SSH 密钥安全**
   - 使用具有有限权限的专用部署密钥
   - 定期轮换 SSH 密钥
   - 永远不要将 SSH 密钥提交到版本控制

2. **环境变量**
   - 在 GitHub Secrets 中存储敏感信息
   - 为开发和生产使用不同的 API 密钥
   - 定期轮换 API 密钥

3. **服务器安全**
   - 保持系统包更新
   - 使用防火墙限制访问
   - 监控服务器日志的可疑活动

### 回滚流程

如果部署失败或出现问题：

```bash
# 连接到服务器
ssh username@your-server-ip

# 导航到项目目录
cd /app/ai-paper-writer

# 回滚到之前的版本
git checkout previous-commit-hash

# 重启服务
pm2 restart ai-paper-writer
```

### 自定义配置

您可以通过修改以下内容来自定义部署流程：

- `.github/workflows/deploy.yml` - 主部署工作流
- `.github/workflows/ci.yml` - CI 和测试工作流
- `部署配置指南.md` - 本文档

### 支持

遇到部署流程问题时：

1. 检查 GitHub Actions 日志获取详细错误信息
2. 查看服务器日志了解应用特定问题
3. 验证所有 GitHub Secrets 配置正确
4. 确保服务器满足所有先决条件

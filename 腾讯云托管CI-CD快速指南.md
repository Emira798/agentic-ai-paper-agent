# 腾讯云托管CI/CD快速指南

## 🎯 一句话总结

**git push → CI测试 → Webhook通知 → 自动部署 → 生产环境更新**

## 🔄 完整流程（30秒理解）

```
你本地开发
    ↓
git add . && git commit -m "更新" && git push origin main
    ↓
【GitHub Actions自动运行CI】
    ├── 代码质量检查（flake8）
    ├── 语法验证
    ├── 真实API测试
    ├── 数据库测试
    └── 覆盖率报告
    ↓
【Webhook自动触发腾讯云】
    ↓
【腾讯云托管检查CI结果】
    ├── ✅ CI通过（绿色）→ 继续部署
    └── ❌ CI失败（红色）→ 取消部署
    ↓
【腾讯云自动部署】
    ├── 拉取最新代码
    ├── 构建Docker镜像
    ├── 停止旧容器
    ├── 启动新容器
    ├── 健康检查
    └── 清理旧镜像
    ↓
🎉 部署完成！你的网站已更新
```

## 🔍 监控要点

### GitHub Actions状态
- ✅ **绿色勾勾**：CI测试通过，允许部署
- ❌ **红色叉叉**：CI测试失败，阻止部署
- ⏳ **黄色时钟**：测试进行中

### 腾讯云托管状态
- ✅ **正常**：服务运行正常
- 🔄 **部署中**：正在更新服务
- ❌ **异常**：服务有问题

## ⚠️ 故障处理

### CI失败（红色叉叉）
- **原因**：代码有错误或测试失败
- **处理**：修复代码，重新提交
- **优点**：坏代码被拦截，不会部署到生产环境

### 部署失败
- **原因**：Docker构建失败或健康检查失败
- **处理**：检查腾讯云托管日志
- **优点**：自动回滚到上一版本

### Webhook失效
- **原因**：连接中断
- **处理**：在腾讯云托管重新授权GitHub
- **预防**：定期检查Webhook状态
